Multi-Agent Reflection 多智能体反思

多个 LLM 扮演不同角色，通过 "生成 → 反馈 → 修正" 循环提升输出质量

从单模型到多智能体的进化

🤖

v1: 单次生成

Prompt → LLM → Response。最简单但质量不可控。

🔄

v2: 自我反思 (Reflexion)

单体 LLM 自己检查自己。但 LLM 倾向于 "和稀泥" -- 很难严格批评自己的输出。

⚙️

v3: 两阶段流水线

Generator 生成 → Critic 批评 → Refiner 修正。角色分离，批评更客观。

🏗️

v4: PEV 架构 (规划-执行-验证)

拆分子任务并行执行，每个经过独立验证。最接近人类解决问题的方式。

⚔️

v5: 辩论 / 委员会

多个 Expert 并行生成，Judge 聚合。强制探索对立观点，消除幻觉。

PEV 架构 (Planner → Executor → Verifier)

🧠 Planner

分析问题，拆解为独立子任务，最终汇总

⚡ Executor

执行各个子任务

Task A
Task B
Task C

✅ Verifier

检查每个子任务输出，不通过则要求重做

1
Planner 收到问题："写一个 Python 爬虫抓取天气数据并分析趋势" → 拆解为: (1) 网页抓取, (2) 数据清洗, (3) 统计分析和可视化
2
Executor 并行执行三个子任务，各自生成代码
3
Verifier 用测试用例验证: Task A (抓取) 通过, Task B (清洗) 通过, Task C (分析) 发现除零错误 → 不通过
4
Executor 收到 Task C 的反馈，修正代码处理边界情况
5
Planner 汇总所有通过的子任务，生成最终完整解决方案

关键洞察

多 Agent 不是简单的 "多跑几次"。核心价值在于 角色分离产生信息多样性: 不同的 Prompt 触发不同的思维路径，Critic/Verifier 消除了单体 LLM "和稀泥" 和 "自欺" 的倾向。
