Token 流经 MoE 层

1
Token 到达 MoE 层 -- 每个 token 的 hidden state 是 [d_model] 向量
2
Router 计算评分 -- 对每个专家计算一个门控分数: gate_score = h @ W_router (d_model 维 → N expert 维)
3
Top-2 选择 (Top-K) -- 只选分数最高的 2 个专家，其余忽略。被选中的专家各自处理这个 token，输出按门控分数加权求和
4
Token 离开 MoE 层 -- 输出维度不变 (d_model)，和 Dense FFN 完全兼容。下一个 MoE 层重复这个过程，可能选不同的专家

关键洞察

8 倍参数，~2 倍推理成本。 MoE 的核心突破: 用稀疏激活打破 "参数量 = 计算量" 的硬性关联。训练时所有专家都得到信号 (通过辅助损失保证负载平衡)，推理时只激活少数几个。
