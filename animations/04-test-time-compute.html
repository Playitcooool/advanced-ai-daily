Test-Time Compute 推理时计算扩展

同样的模型，给它更多思考时间，性能显著提升

训练时扩展

更多的数据 + 参数 + FLOPs

推理时扩展

更多的采样 + 思考 + 搜索

四种核心方法

Best-of-N

生成 N 个答案，用奖励函数选最好的。N 越大找到好答案概率越高，但边际效益递减。

N=1→N=100: 30% 提升到 55%+ 正确率

Self-Correction

模型自己检查、批评、修正答案。需要可靠的自评估 Prompt。知识缺失时无效。

适合: 模型知道答案但不确定性高的情况

Tree of Thoughts

结构化思维搜索: 分支 K 个 → 评估 → 保留 Top-B → 继续。比线性 CoT 探索更多推理路径。

Beam Size=3, Depth=4 效果显著

Verifier-Guided

大量采样 + 代码执行器验证。AlphaCode 方法。只要有客观验证器，效果最好。

128 个候选: 编程竞赛 Codeforces 前 50%

关键洞察

训练时扩展和推理时扩展是两条互补的 Scaling Law。未来模型 = 更好的基础 + 更多的推理预算。OpenAI o1 和 DeepSeek-R1 本质上都是训练模型在推理时 "花更多时间思考"。
